# Plan 0005 — JA/EN internationalization + English code comments

Status: IMPLEMENTED 2026-06-21 (branch `feat/i18n-ja-en`). Verified by `tsc --noEmit` + `npm run build` (both clean). Real-world check (toggle JA/EN in the running app) still recommended.

## Decisions
- **Language switcher: Home only.** A JA/EN toggle in the Home screen's top bar. No dedicated first-launch screen. Default **Japanese**; choice persisted to `localStorage`.
- **Comments: all translated to English.** Every Japanese code comment (and dev-facing debug/log string) across the codebase is now English. This was in scope per the user's request, separate from the user-facing toggle.

## Pattern (chosen for parallel-safe conversion)
Each component holds its **own** bilingual strings locally — no shared dictionary file, so many files could be converted in parallel without contention:
```ts
const STRINGS = { ja: { key: '日本語' }, en: { key: 'English' } } as const;
// in component:
const language = useDrivingStore((s) => s.language);
const t = STRINGS[language];
// use t.key
```
Store additions: `language: 'ja' | 'en'` + `setLanguage` (persisted to `localStorage`, default `ja`). See `src/components/ui/HomeScreen.tsx` for the reference implementation.

## Scope completed
- Bilingual UI: HomeScreen (+ switcher), AuthScreen, FeedbackScreen, HistoryScreen (incl. `getLessonName` + date locale), ClientApp (mission briefings, pause overlay), PauseMenu, Dashboard, TutorialIndicators, TutorialScreen.
- English comments / debug strings: VisionController, footPedalRecognition, oneEuroFilter, Car, Scene, RoadProps, ThreeModelLoader, TrafficSystem, all simulation/objects, useRegisterCheckpoint, debug/page, MissionController, store.ts. (debug/page, an internal page, was translated directly to English, not bilingual.)

## Feedback messages — RESOLVED (2026-06-21)
The per-run **KAIZEN feedback messages** (`store.ts` `calculateMissionResult`) and the **"OK!" checkpoint toasts** (`Car.tsx`) are now bilingual, driven by `language` from the store. Approach: **type-and-language driven** rather than label-driven — English mode uses wording based on `cp.type` (e.g. "You ignored a required stop"), Japanese mode keeps the specific `cp.label` text. This avoided changing the `MissionCheckpoint.label` type across the codebase. The Japanese labels in `MissionController.tsx` are therefore the intended JA-mode display text, not untranslated leftovers. English mode (including the saved `feedbackSummary`) is now fully English. Verified by `tsc --noEmit` + `npm run build`.
